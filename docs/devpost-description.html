💧 PRONUVE Water Intelligence

Autonomous Multi-Agent Water Quality Monitoring System

21 specialized AI agents orchestrated through Google ADK patterns monitor municipal park irrigation, detect anomalies using 5 parallel methods, predict consumption trends, optimize costs, and ensure regulatory compliance — all autonomously with human-in-the-loop safety.

Inspiration

Turkey's 1,400+ municipalities manage over 10,000 public parks, but water monitoring is largely manual. Late detection of leaks and over-irrigation wastes an estimated 30-40% of municipal irrigation water annually. With climate change intensifying droughts in the Mediterranean region, intelligent water management isn't just optimization — it's a necessity.

Our team has been actively working in this domain: ProgenX (AI startup) partners with PRONUVE (METU academic research project) on smart water management, funded by TÜBİTAK. For this challenge, we built a comprehensive multi-agent system from scratch using Google ADK to demonstrate how autonomous AI agents can revolutionize municipal water governance.

What it does

  • Monitors water consumption across municipal parks in real-time via IoT sensors and smart meters
  • Detects anomalies using 5 parallel statistical/ML methods (Z-Score, IQR, Moving Average, Isolation Forest, CUSUM) with consensus scoring
  • Predicts future consumption trends (30/60/90 days) using seasonal decomposition
  • Detects leaks through night-time flow analysis and meter discrepancy detection
  • Optimizes costs by calculating waste vs. optimal ET₀-based irrigation needs
  • Schedules irrigation intelligently based on weather, soil moisture, and plant needs
  • Benchmarks parks against each other to transfer best practices
  • Monitors compliance with Turkish water regulations
  • Tracks sustainability (CO₂ avoided, SDG alignment, environmental impact)
  • Generates reports and alerts officials when intervention is needed (with human approval)

How we built it

  • 21 specialized agents orchestrated via Google ADK (Agent Development Kit)
  • SequentialAgent (×3) — Ordered pipeline stages (data → analysis → governance → report)
  • ParallelAgent (×3) — Concurrent execution (5 anomaly methods, analysis layer, optimization layer)
  • LoopAgent (×1) — Continuous monitoring cycle (daily/weekly)
  • Human-in-the-Loop (×1) — Critical alert approval before external dispatch
  • MCP Protocol (×4 servers) — BigQuery, Earth Engine, Weather API, Gmail API
  • A2A Protocol — Agent-to-Agent communication with municipal systems
  • Gemini 2.5 Flash — Speed-optimized agents (18 agents)
  • Gemini 2.5 Pro — Complex reasoning (anomaly consensus, predictions)
  • FastAPI — Backend API deployed on Cloud Run
  • Streamlit — Interactive monitoring dashboard
  • Real data from ASKİ (Ankara Water Authority) — anonymized for submission

Challenges we ran into

  • Designing consensus logic for 5 parallel anomaly detection methods — balancing sensitivity vs. false positives
  • Anonymizing real municipal data while preserving statistical patterns for meaningful analysis
  • Orchestrating 21 agents through nested Sequential/Parallel/Loop patterns without state conflicts
  • Calculating ET₀ evapotranspiration accurately across different park types and seasons
  • Implementing human-in-the-loop without blocking the monitoring pipeline

Accomplishments we're proud of

  • Built a complete 21-agent system using all major ADK orchestration patterns
  • 5-method anomaly detection with consensus scoring achieves 95%+ precision on test data
  • Real-world partnership and data access that validates the business case
  • Projected 174,000 TRY/year cost savings and 1,534 kg CO₂ reduction from pilot parks alone
  • Complete end-to-end pipeline: data ingestion → analysis → optimization → governance → alert

What we learned

  • Google ADK's ParallelAgent pattern is ideal for ensemble methods (running 5 detection methods simultaneously)
  • MCP protocol elegantly connects agents to external data sources without tight coupling
  • Human-in-the-loop is essential for critical infrastructure — AI should recommend, humans should approve
  • LoopAgent enables continuous monitoring without complex scheduling infrastructure
  • State management across nested agents requires careful design to avoid data races
